# 一、项目概述

## 1\.1 项目背景

传统校园课程管理依赖线下人工登记、纸质档案，存在课程信息查找慢、公告下发不及时、学生无法自主查看课程、管理员维护成本高等痛点。为实现校园课程数字化管理，搭建一套**Web 学校课程管理系统**，区分管理员后台与学生前台两大端口，完成课程分类、课程信息、校园公告、学生收藏、账号权限一体化管理。

## 1\.2 项目目标

1. 搭建前后端分离架构，后端 SpringBoot 提供 RESTful 接口，Vue 实现前台学生页面与后台管理页面；

2. 区分两种用户角色：**管理员**、**学生**，权限隔离；

3. 学生前台：首页轮播、公告浏览、课程分类检索、课程详情查看、课程收藏、个人中心（修改密码、我的收藏）、账号注册登录；

4. 管理员后台：公告管理、师生用户管理、课程分类管理、课程信息 CRUD、轮播图管理；

5. 实现图片上传、分页查询、条件搜索、数据新增 / 编辑 / 删除、收藏功能；

6. 统一异常处理、登录 Token 鉴权，保证系统数据安全。

## 1\.3 用户角色说明

1. **管理员**：后台系统登录，拥有全部数据管理权限，维护课程、公告、师生账号、轮播图；

2. **学生**：前台页面注册登录，仅可浏览、收藏课程，管理个人账号与收藏列表，无数据修改权限。

# 二、系统需求分析

## 2\.1 功能性需求

### （1）登录注册模块

- 登录：区分管理员 / 学生两种身份登录，校验账号密码；

- 学生注册：填写学号、密码、姓名、头像、性别、手机号完成账号注册；

- 退出登录：清除本地 Token，跳转登录页；

- 修改密码：学生在个人中心校验原密码，设置新密码。

### （2）学生前台功能（对应截图 1、2、3、4、9）

1. 首页：顶部轮播图展示校园宣传图，侧边导航栏（首页、公告信息、课程信息）；

2. 公告信息：查看管理员发布的全部校园公告；

3. 课程信息页：左侧课程分类筛选框、顶部课程名称搜索框，卡片式展示课程；

4. 课程详情：展示课程封面、分类、课时、上课时间、授课教师，支持收藏 / 取消收藏；

5. 个人中心：修改密码、我的收藏管理（查看已收藏课程）；

### （3）管理员后台功能（对应截图 5、6、7、8）

1. 公告信息管理：公告新增、编辑、删除、分页列表；

2. 用户管理：教师管理、学生管理，对师生账号进行维护；

3. 课程信息管理

    - 课程分类：新增、修改、删除课程分类；

    - 课程信息：分页列表、按名称搜索、新增课程弹窗（上传封面、选择分类、填写课时 / 教师 / 上课时间）、详情查看、编辑、删除、查看课程评论；

4. 轮播图管理：维护前台首页轮播图片；

## 2\.2 非功能性需求

1. **性能**：列表分页加载，单页数据不超过 10 条，接口响应≤500ms；

2. **安全性**：基于 Token 登录鉴权，未登录用户禁止访问业务页面；密码加密存储；

3. **易用性**：页面操作简洁，弹窗表单校验必填项，搜索、分页直观；

4. **兼容性**：Vue 页面适配主流浏览器（Chrome、Edge）；

5. **可扩展性**：模块化开发，后续可拓展课程评论、选课、成绩模块。

# 三、技术栈选型

## 3\.1 后端技术栈（SpringBoot）

1. 核心框架：SpringBoot 2\.7\.x，快速构建 Web 服务，内置 Tomcat；

2. ORM 持久层：MyBatis\-Plus 3\.5，简化 CRUD、分页插件、逻辑删除；

3. 数据库：MySQL 8\.0，字符集 utf8mb4，支持图片路径、中文存储；

4. 权限认证：JWT，无状态 Token 登录校验，区分管理员 / 学生权限；

5. 文件上传：SpringMVC 文件上传，本地存储课程封面、头像、轮播图；

6. 工具类：Hutool 工具包、FastJSON2 序列化；

7. 接口文档：Knife4j（Swagger），可视化在线接口调试；

8. 全局处理：统一返回结果封装、全局异常处理器、跨域 CORS 配置；

9. 构建工具：Maven 3\.6\+。

## 3\.2 前端技术栈（Vue2/Vue3）

1. 核心框架：Vue3 \+ Vue CLI，前后端分离；

2. UI 组件库：Element Plus（管理员后台）、原生自定义页面（学生前台）；

3. 路由：VueRouter，区分前台路由 / 后台路由，路由守卫拦截未登录访问；

4. 状态管理：Pinia，存储登录用户信息、全局 Token；

5. 网络请求：Axios 二次封装请求拦截、响应拦截，自动携带 Token、统一错误提示；

6. 工具：Element Plus 上传组件（头像 / 课程封面）、分页组件、弹窗组件；

7. 开发工具：VS Code。

## 3\.3 开发 \& 运维工具

1. 后端开发：IDEA 2022\+；

2. 数据库可视化：Navicat；

3. 版本控制：Git；

4. 服务器部署：CentOS7、JDK1\.8、Nginx 反向代理。

# 四、系统整体架构

采用标准**前后端分离三层架构**，前端与后端通过 HTTP JSON 接口交互，完全解耦。

## 4\.1 前端分层

1. **视图层 View**：前台学生页面、管理员后台页面、弹窗表单；

2. **路由层 Router**：路由守卫，校验 Token 登录状态，区分前台 / 后台访问权限；

3. **网络请求层 Axios**：统一封装请求头、Token 携带、响应码拦截（401 未登录跳转登录页）；

4. **状态管理 Pinia**：存储当前登录用户、token、全局基础参数；

5. **工具层 Utils**：时间格式化、文件上传处理、表单校验、弹窗提示。

## 4\.2 后端分层（标准 MVC 分层）

```Plain Text
com.school.course
├── config        全局配置类
│   ├── CorsConfig.java      跨域配置
│   ├── JwtInterceptor.java  Token拦截器
│   ├── MybatisPlusConfig.java 分页插件
│   └── Knife4jConfig.java    接口文档
├── controller    控制层（接收前端请求）
│   ├── AdminController.java 管理员登录
│   ├── StudentController.java学生登录注册
│   ├── CourseController.java课程/分类接口
│   ├── NoticeController.java公告接口
│   ├── UploadController.java文件上传
│   └── CollectController.java课程收藏
├── service       业务逻辑层
│   └── impl      业务实现类
├── mapper        MyBatis-Plus持久层接口
├── entity        数据库实体类
├── dto           前后端数据传输对象（入参、分页出参）
├── util          工具类（Jwt工具、统一返回工具、文件工具）
├── exception     全局异常处理、自定义业务异常
└── CourseApplication.java 启动类
```

## 4\.3 数据交互流程

1. 用户在 Vue 页面发起操作（登录、查询课程、新增课程）；

2. Axios 拦截器自动在请求头携带 Token；

3. 请求发送至 SpringBoot 后端接口；

4. JWT 拦截器校验 Token 有效性，未登录直接返回 401；

5. Controller 接收参数，调用 Service 处理业务逻辑；

6. Service 调用 Mapper 操作 MySQL 数据库；

7. 处理完成封装统一 JSON 结果返回前端；

8. Axios 响应拦截统一处理成功 / 失败提示，渲染页面数据。

## 模块 1：登录注册模块

### 5\.1 系统登录页（管理员 / 学生共用登录入口）

页面功能：

1. 输入账号、密码，下拉选择用户类型：管理员 / 学生；

2. 表单非空校验，错误提示；

3. 登录成功存储 Token 至本地存储，跳转对应页面（管理员→后台首页，学生→前台首页）；

1. 学号唯一校验，重复学号提示注册失败；

# 七、核心业务流程

## 7\.1 学生登录浏览课程完整流程

1. 学生访问前台页面，未登录自动跳转登录页；

2. 无账号点击注册，填写信息完成学生账号注册；

3. 输入学号密码登录，后端校验账号密码，生成 JWT Token 返回前端；

4. 前端存储 Token，跳转前台首页加载轮播图；

5. 侧边栏点击课程信息，请求后端全部分类数据，左侧渲染分类筛选栏；

6. 输入课程名称搜索 / 点击分类筛选，后端分页返回课程列表；

7. 点击课程卡片，请求课程详情接口，展示课程全部信息；

8. 点击收藏按钮，后端新增收藏记录；

9. 进入个人中心，查看我的收藏列表，可取消收藏；

10. 修改密码：校验原密码，更新数据库学生密码。

## 7\.2 管理员新增课程流程

1. 管理员后台登录，校验账号密码获取管理员 Token；

2. 左侧菜单进入课程信息管理 → 课程信息列表；

3. 点击新增按钮弹出课程表单弹窗；

4. 上传课程封面图片，后端接收文件返回图片 URL；

5. 填写课程名称、选择课程分类、课时、上课时间、教师工号；

6. 点击提交，后端校验必填字段，数据插入 course 课程表；

7. 列表自动刷新展示新增课程，支持编辑、删除、查看详情。

## 7\.3 Token 鉴权安全流程

1. 登录成功后端生成带用户 ID、用户类型（管理员 / 学生）的 JWT 令牌；

2. 前端所有业务请求 Axios 拦截器自动携带 Token 到请求头；

3. 后端全局 JwtInterceptor 拦截所有接口；

4. 未携带 Token/Token 过期 / Token 非法，统一返回 code=401；

5. 前端响应拦截捕获 401，清除本地 Token，强制跳转登录页面。

# 十、项目总结

本 Web 学校课程管理系统采用 SpringBoot\+Vue 标准前后端分离架构，精准贴合校园课程数字化管理需求，划分学生前台浏览端口与管理员后台数据管理端口，覆盖课程分类、课程信息、校园公告、学生收藏、账号管理、图片上传等全部核心业务。
系统分层清晰、代码模块化、基于 JWT 实现登录权限控制，使用 MyBatis\-Plus 简化数据库操作，分页、搜索、文件上传等通用功能封装完善，易于二次开发与功能拓展。同时页面设计贴合校园场景，操作流程简洁直观，满足学校课程日常管理与学生线上选课查看需求，具备完整落地商用能力。
